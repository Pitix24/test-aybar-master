# Resumen de cambios en Tickets

Se realizaron mejoras para que el proceso de tickets sea mas ordenado y facil de usar.

## Que se corrigio

- Ahora el area elegida controla mejor la seleccion del gestor.
- Si un area no tiene gestores disponibles, el sistema lo muestra con un aviso claro.
- Al crear tickets hijos, el sistema tambien respeta el area y solo muestra opciones validas.
- En la derivacion de tickets, el gestor y el area quedan correctamente vinculados.
- Si existe un ticket con datos antiguos o inconsistentes, el sistema intenta corregirlo al editarlo.

## Que gana el usuario

- Menos errores al crear o mover tickets.
- Mensajes mas claros cuando falta una configuracion.
- Mayor orden entre el area del ticket y la persona encargada.
- Menos riesgo de que un ticket quede asignado a una combinacion incorrecta.

## Resultado esperado

El sistema ahora ayuda a guiar mejor al usuario durante la creacion, edicion, derivacion y creacion de tickets hijos, evitando asignaciones que no corresponden y mostrando avisos comprensibles cuando algo no esta disponible.
